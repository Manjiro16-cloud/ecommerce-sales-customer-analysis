# 电商销售与客户经营分析

## 项目简介

本项目使用 UCI Online Retail II 公开零售订单数据，模拟数据运营岗位的日常分析场景。通过 Python 完成数据检查、清洗和指标汇总，并使用 Excel/WPS 制作经营分析看板。

## 项目目标

- 分析销售额、订单量和客户数量变化
- 识别重点月份、重点市场和高销售额商品
- 完成基础客户分层
- 根据数据提出经营建议
- 记录数据限制，避免超出数据范围得出结论

## 数据概况

- 原始明细：525,461条
- 时间范围：2009年12月—2010年12月
- 国家或地区：40个
- 有效销售明细：504,730条

## 数据处理

1. 使用 Python 和 pandas 检查数据质量。
2. 删除 6,865 条完全重复记录。
3. 将取消订单、非正数量、非正价格等 13,866 条异常记录单独保存。
4. 生成销售额、年月、星期和交易小时等分析字段。
5. 商品排行中排除人工调整、邮费和手续费等非普通商品交易项。
6. 缺少客户编号的记录保留在整体销售分析中，但不用于客户分层。

## 核心结果

- 销售额：£10,271,762.65
- 订单数：20,951
- 客户数：4,312
- 平均订单金额：£490.28
- 英国贡献约85.8%的销售额
- 2010年11月销售额约£1.46M
- 活跃客户贡献约£5.02M

## 数据限制

- 部分销售明细缺少客户编号，客户分析不能覆盖全部销售。
- 2009年12月和2010年12月不是完整月份。
- 数据不包含成本、流量和库存，无法直接评价利润、转化率和库存周转。
- 本项目使用公开数据，不代表企业实习经历。
